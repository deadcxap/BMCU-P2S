# Документация: `MC_PULL_calibration` (`src/MC_PULL_calibration.cpp/.h`)

## Назначение
Модуль выполняет первичную и сервисную калибровку analog pull-датчиков по каналам 0..3:
- вычисляет `MC_PULL_V_OFFSET` (центрирование в 1.65 В);
- измеряет `MC_PULL_V_MIN/MAX`;
- определяет `MC_PULL_POLARITY`;
- вычисляет порог `MC_DM_KEY_NONE_THRESH` по key-каналу;
- сохраняет калибровку в Flash.

См. также:
- [Motion control](./motion_control_ru.md)

## Публичный интерфейс
- `void MC_PULL_calibration_boot()` — запуск логики калибровки при старте.
- `void MC_PULL_calibration_clear()` — очистка калибровки и motion-настроек во Flash.

## Входные зависимости
- `ADC_DMA` — сырые analog-значения.
- `Motion_control` — общие массивы калибровки/состояний.
- `Flash_saves` — чтение/запись калибровочных структур.
- `time_hw` + LED API — тайминги и визуальная подсказка пользователю.

## Внутренние этапы калибровки
## 1) Boot: попытка загрузки из Flash
В `MC_PULL_calibration_boot()` сначала читается Flash (`Flash_MC_PULL_cal_read`).
Если данные валидны, модуль просто применяет их к глобальным массивам и завершает работу.

## 2) Если Flash пуст/невалиден — автоматический захват baseline
- Несколько циклов стабилизации ADC.
- Детект физически вставленных каналов (`MC_PULL_detect_channels_inserted`).
- Усреднение `N=90` измерений для `pull_raw` и `key_raw`.
- Вычисление:
  - `MC_PULL_V_OFFSET[ch] = 1.65 - center_raw[ch]`;
  - `MC_DM_KEY_NONE_THRESH[ch]` через `dm_key_none_threshold_from_idle(...)` (с ограничением диапазона порога).

## 3) Захват экстремумов (min/max + polarity)
Для каждого подключённого канала:
- `capture_first_extreme_wait_release(...)`:
  - ждёт первое уверенное отклонение от центра;
  - определяет знак полярности (какая сторона является «рабочей»);
  - ждёт возврат к центру.
- `capture_second_extreme_wait_release(...)`:
  - захватывает второй экстремум с учётом найденной полярности.
- `capture_minmax_one_ch_event(...)`:
  - объединяет обе стадии;
  - применяет защитные минимальные рамки по диапазону.

## 4) Сохранение
После расчёта всех каналов:
- `Flash_MC_PULL_cal_write_all(...)` — офсеты/минимумы/максимумы/полярность;
- `Motion_control_save_dm_key_none_thresholds()` — пороги DM key;
- LED-индикация успеха/ошибки (жёлтый/красный blink).

## Служебные функции
- `adc_pull_raw_ch(...)`, `adc_key_raw_ch(...)` — маппинг ADC-индексов на логические каналы.
- `cal_apply_polarity(...)` — приведение к единой «физической» оси.
- `dm_key_round_up_to_centi(...)` — квантование в сотые вольта.

## Параметры/тайминги
- `CAL_PRESS_DELTA_V = 0.1V` — порог «нажатия/смещения»;
- `CAL_CENTER_EPS_V = 0.02V` — зона «вернулись в центр»;
- `CAL_STABLE_MS = 200ms` — сколько держать центр для фиксации;
- `CAL_TIMEOUT_MS = 30000ms` — таймаут фазы захвата.

## Поведение для отсутствующего канала
Если канал физически не вставлен:
- ставятся безопасные дефолты (`V_MIN=1.55`, `V_MAX=1.75`, `POLARITY=1`);
- `MC_DM_KEY_NONE_THRESH` задаётся нейтрально;
- интерактивный захват экстремумов пропускается.
